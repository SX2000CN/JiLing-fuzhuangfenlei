# 部署包说明（当前仓库实际）

更新时间：2026-04-07

## 1. 当前状态

1. 仓库内存在打包脚本：`build/make_installer.bat`。
2. 当前仓库未内置可直接分发的安装包产物（例如 setup.exe）。
3. 打包结果依赖本机环境与脚本配置，需在本地执行脚本生成。

## 2. 可用启动方式（开发/测试）

1. 新架构入口：`python run_new_gui.py`
2. 稳定新界面：`launchers/启动-新界面.bat`
3. 传统界面：`launchers/快速启动-传统界面.bat`
4. CLI：`launchers/run_classify.bat`

## 3. 打包前检查清单

1. `.conda/python.exe` 可用。
2. `config/`、`models/`、`fonts/` 目录完整。
3. 入口脚本与文档一致。
4. `python -m pytest tests -q` 通过。

## 4. 打包流程（仓库内可见）

1. 进入项目根目录。
2. 执行 `build/make_installer.bat`。
3. 检查脚本输出目录是否生成可执行产物。

本文件仅记录当前仓库可确认事实，不再保留历史安装包大小、版本号等不可复验信息。
