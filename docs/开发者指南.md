# 开发者指南（以当前仓库为准）

更新时间：2026-04-07

## 1. 入口约定（最新版）

1. 稳定新界面：`run_new_gui.py`（默认）。
2. 模块化预览：`run_new_gui.py --modular`。
3. 传统界面：`run_new_gui.py --traditional`。
4. CLI：`launchers/run_classify.bat` 或 `python src/cli/classify_cli.py --help`。

## 2. 启动脚本约定

`launchers/` 下 GUI 脚本统一调用 `run_new_gui.py`，不再直接调用多个 GUI 实现文件。

## 3. 代码结构（实际）

```text
src/
├─ cli/
│  └─ classify_cli.py
├─ core/
│  ├─ model_factory.py
│  ├─ pytorch_classifier.py
│  └─ pytorch_trainer.py
├─ gui/

│  ├─ native_ui.py
│  ├─ main_window.py
│  ├─ components/

│  ├─ theme/
│  ├─ workers/
│  ├─ widgets/
│  └─ animations/
└─ utils/
   ├─ config_manager.py
   ├─ image_utils.py
   └─ logging_utils.py
```

## 4. GUI 三实现定位

| 文件 | 定位 | 当前状态 |
|---|---|---|
| `src/gui/native_ui.py` | 稳定新界面实现 | 当前默认主线 |
| `src/gui/app.py` | 模块化新架构实现 | 预览入口，待完整回归 |
| `src/gui/main_window.py` | 传统界面实现 | 历史兼容 |

## 5. 常用命令

```bash
python run_new_gui.py
python run_new_gui.py --modular
python run_new_gui.py --traditional
python src/cli/classify_cli.py --help
python -m pytest tests -q
```

## 6. 开发约束

1. 用户可见文案保持中文。
2. GUI 耗时任务必须走线程/worker，不可阻塞主线程。
3. 不新增“并行第四套 GUI”；先做现有三套收敛。
4. 入口变更必须同步更新 README 与 docs 主文档。
