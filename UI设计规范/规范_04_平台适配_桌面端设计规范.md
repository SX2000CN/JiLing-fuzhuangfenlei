# 桌面端设计规范

## 概述

桌面端应用面向专业用户，需要支持复杂工作流、高效操作和深度系统集成。本规范涵盖 Windows、macOS 和 Linux 桌面应用的设计标准。

## 设计原则

- **效率至上**：支持专业用户高效工作
- **功能完整**：提供完整的功能集
- **系统融合**：与操作系统深度集成
- **多任务支持**：支持并行工作流

## 窗口设计

### 窗口尺寸

| 尺寸 | 宽度 | 高度 | 用途 |
|------|------|------|------|
| 最小 | 800px | 600px | 基础功能 |
| 默认 | 1280px | 800px | 标准使用 |
| 推荐 | 1440px | 900px | 舒适体验 |

### 窗口结构

```
┌─────────────────────────────────────────────────┐
│  ● ● ●   应用名称                    ─ □ ✕     │  ← 标题栏
├─────────────────────────────────────────────────┤
│  文件  编辑  视图  帮助                         │  ← 菜单栏
├─────────────────────────────────────────────────┤
│  [工具栏按钮]  [工具栏按钮]  [工具栏按钮]       │  ← 工具栏
├──────────┬──────────────────────────────────────┤
│          │                                      │
│  侧边栏  │              主内容区                │
│          │                                      │
├──────────┴──────────────────────────────────────┤
│  状态信息                              缩放 100%│  ← 状态栏
└─────────────────────────────────────────────────┘
```

### 窗口行为

| 功能 | 支持 |
|------|------|
| 调整大小 | ✓ |
| 最大化/还原 | ✓ |
| 最小化 | ✓ |
| 全屏 | ✓ |
| 记住位置 | ✓ |
| 多窗口 | 可选 |

## 平台差异

### macOS 特性

| 元素 | 规范 |
|------|------|
| 窗口控制 | 左上角（红黄绿） |
| 菜单栏 | 系统顶部全局菜单 |
| 字体 | SF Pro |
| 圆角 | 较大 |
| 触控板 | 多点手势支持 |

### Windows 特性

| 元素 | 规范 |
|------|------|
| 窗口控制 | 右上角（最小化、最大化、关闭） |
| 菜单栏 | 窗口内 |
| 字体 | Segoe UI |
| 圆角 | Windows 11 支持 |
| 任务栏 | 进度显示、跳转列表 |

### Linux 特性

| 元素 | 规范 |
|------|------|
| 窗口控制 | 遵循桌面环境 |
| 菜单栏 | 窗口内或全局 |
| 字体 | 系统默认 |
| 主题 | 支持系统主题 |

## 菜单设计

### 菜单栏

| 标准菜单 | 内容 |
|----------|------|
| 文件 | 新建、打开、保存、导出、退出 |
| 编辑 | 撤销、重做、剪切、复制、粘贴 |
| 视图 | 缩放、布局、侧边栏 |
| 帮助 | 文档、关于、检查更新 |

### 右键菜单

| 规范 | 值 |
|------|-----|
| 最大项数 | 10-12 项 |
| 分组 | 使用分隔线 |
| 快捷键 | 右对齐显示 |
| 子菜单 | 最多一层 |

### 快捷键

| 操作 | Windows | macOS |
|------|---------|-------|
| 保存 | Ctrl+S | Cmd+S |
| 撤销 | Ctrl+Z | Cmd+Z |
| 复制 | Ctrl+C | Cmd+C |
| 粘贴 | Ctrl+V | Cmd+V |
| 查找 | Ctrl+F | Cmd+F |
| 全选 | Ctrl+A | Cmd+A |
| 关闭 | Ctrl+W | Cmd+W |
| 退出 | Alt+F4 | Cmd+Q |

## 性能考虑

| 方面 | 要求 |
|------|------|
| 启动时间 | < 3秒 |
| 响应时间 | < 100ms |
| 内存占用 | 合理范围 |
| CPU 使用 | 空闲时低 |

## 无障碍要求

| 要求 | 实现 |
|------|------|
| 屏幕阅读器 | 支持系统辅助功能 |
| 键盘导航 | 完全可操作 |
| 高对比度 | 支持系统主题 |
| 缩放 | 支持系统缩放 |

## 最佳实践

### ✅ 推荐
- 遵循平台设计规范
- 提供丰富快捷键
- 支持多窗口工作
- 记住用户偏好

### ❌ 避免
- 忽视平台差异
- 缺少键盘支持
- 阻塞主线程
- 忽视系统集成
