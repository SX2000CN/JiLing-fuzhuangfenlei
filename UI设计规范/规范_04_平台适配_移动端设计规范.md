# 移动端设计规范

## 概述

移动端设计需要考虑触控交互、有限屏幕空间和移动使用场景。本规范涵盖 iOS、Android 原生应用及移动网页的设计标准。

## 设计原则

- **触控优先**：为手指而非鼠标设计
- **简洁聚焦**：每屏专注一个任务
- **快速响应**：即时反馈用户操作
- **情境感知**：适应移动使用场景

- **设计稿宽度**：375px（iOS）/ 360px（Android）
- **适配方式**：等比缩放 + 安全区域

## 性能优化

### 加载优化

| 策略 | 应用 |
|------|------|
| 骨架屏 | 首屏加载 |
| 懒加载 | 图片、列表 |
| 预加载 | 下一页内容 |
| 缓存 | 离线数据 |

### 交互优化

- 触觉反馈
- 60fps 动画
- 避免阻塞主线程
- 手势响应 < 100ms

## 无障碍要求

| 要求 | 实现 |
|------|------|
| VoiceOver/TalkBack | 语义化标签 |
| 动态字体 | 支持系统字号 |
| 对比度 | 4.5:1 以上 |
| 触控目标 | 44×44px 以上 |

## 最佳实践

### ✅ 推荐
- 遵循平台设计规范
- 优化单手操作
- 提供触觉反馈
- 支持深色模式

### ❌ 避免
- 照搬网页设计
- 触控目标过小
- 忽视安全区域
- 过度使用手势
