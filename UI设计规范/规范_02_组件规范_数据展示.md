# 数据展示

## 概述

数据展示组件用于呈现和组织信息，帮助用户快速浏览、理解和操作数据。包括表格、列表、卡片、标签等组件。

## 设计原则

- **可扫描**：用户能快速浏览找到所需信息
- **有层次**：重要信息突出显示
- **可操作**：支持排序、筛选、选择等操作
- **响应式**：适应不同屏幕尺寸

## 表格（Table）

### 结构

```
┌──────────────────────────────────────────────────┐
│ □  列标题1 ↕    列标题2      列标题3      操作   │  ← 表头
├──────────────────────────────────────────────────┤
│ □  数据1         数据2        数据3       编辑   │  ← 数据行
│ □  数据1         数据2        数据3       编辑   │
│ □  数据1         数据2        数据3       编辑   │
├──────────────────────────────────────────────────┤
│                           共 100 条  < 1 2 3 >   │  ← 分页
└──────────────────────────────────────────────────┘
```

## 平台差异

### 网页端
- 表格支持完整功能
- 支持复杂交互（拖拽排序等）
- 可使用更高信息密度

### 移动端
- 优先使用列表和卡片
- 表格需水平滚动或响应式布局
- 简化操作，使用滑动操作
- 分页改为无限滚动

### 桌面端
- 可使用紧凑密度
- 支持键盘导航
- 右键菜单操作

## 无障碍考虑

- 表格使用 `<table>` 语义标签
- 提供表头 `<th scope="col">`
- 可排序列使用 `aria-sort`
- 选择状态使用 `aria-selected`
- 进度条使用 `role="progressbar"`

## 最佳实践

### ✅ 推荐
- 大数据量使用虚拟滚动
- 提供空状态引导
- 支持键盘操作表格
- 复杂表格提供筛选功能

### ❌ 避免
- 移动端强制使用复杂表格
- 单元格内容过长无处理
- 无加载状态
- 忽略空状态设计
