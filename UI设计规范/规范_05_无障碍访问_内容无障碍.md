# 内容无障碍

## 概述

内容无障碍确保所有用户都能理解和获取信息内容。本规范涵盖文本内容、多媒体、文档结构和语义化标记的无障碍要求。

## 设计原则

- **清晰易懂**：使用简洁明了的语言
- **结构化**：内容有清晰的层级结构
- **多模态**：提供多种内容形式
- **语义化**：使用正确的语义标记

## 文本内容

### 语言声明

```html
<!-- 页面语言 -->
<html lang="zh-CN">

<!-- 局部语言变化 -->
<p>这是中文内容，<span lang="en">this is English</span>。</p>
```

### 阅读级别

| 目标 | 建议 |
|------|------|
| 通用内容 | 初中阅读水平 |
| 专业内容 | 提供简化版本 |
| 术语 | 提供定义或解释 |

### 写作指南

| 原则 | 说明 |
|------|------|
| 简洁 | 避免冗长句子 |
| 主动语态 | 优先使用主动语态 |
| 常用词汇 | 避免生僻词 |
| 解释缩写 | 首次使用时展开 |

## 标题结构

### 层级规范

```html
<h1>页面标题</h1>
  <h2>主要章节</h2>
    <h3>子章节</h3>
    <h3>子章节</h3>
  <h2>主要章节</h2>
    <h3>子章节</h3>
```

### 标题要求

| 要求 | 说明 |
|------|------|
| 唯一 h1 | 每页一个 h1 |
| 不跳级 | h2 后是 h3，不是 h4 |
| 描述性 | 标题描述内容 |
| 简洁 | 标题简短明了 |

## 链接文本

### 有意义的链接

| ❌ 避免 | ✅ 推荐 |
|--------|--------|
| 点击这里 | 查看产品详情 |
| 更多 | 更多关于无障碍的文章 |
| 链接 | 下载用户手册 (PDF, 2MB) |

### 链接上下文

```html
<!-- 如果链接文本不够清晰，使用 aria-label -->
<a href="/products" aria-label="查看所有产品列表">
  查看更多
</a>

<!-- 或使用 aria-describedby -->
<p id="product-desc">iPhone 15 Pro</p>
<a href="/iphone" aria-describedby="product-desc">了解更多</a>
```

## 图像内容

### 替代文本

| 图像类型 | alt 处理 |
|----------|----------|
| 信息性 | 描述图像内容 |
| 装饰性 | alt="" |
| 功能性 | 描述功能/目的 |
| 文字图像 | 包含图中文字 |

### 复杂图像

```html
<!-- 图表、信息图等复杂图像 -->
<figure>
  <img src="chart.png" alt="2024年销售趋势图">
  <figcaption>
    图表显示2024年销售额逐月增长，
    从1月的100万增长到12月的250万。
  </figcaption>
</figure>
```

### 图像文字

| 原则 | 说明 |
|------|------|
| 避免 | 尽量不用图像显示文字 |
| 必要时 | 提供完整替代文本 |
| Logo | 可以使用图像 |

## 多媒体内容

### 视频

| 要求 | 说明 |
|------|------|
| 字幕 | 同步字幕（CC） |
| 音频描述 | 描述视觉内容 |
| 文字稿 | 完整文字版本 |
| 控制 | 播放/暂停/音量 |

### 音频

| 要求 | 说明 |
|------|------|
| 文字稿 | 完整文字版本 |
| 控制 | 播放/暂停/音量 |
| 自动播放 | 避免或可停止 |

### 字幕规范

| 元素 | 要求 |
|------|------|
| 同步 | 与音频同步 |
| 完整 | 包含所有对话 |
| 音效 | 描述重要音效 |
| 说话人 | 标识不同说话人 |

## 表格内容

### 数据表格

```html
<table>
  <caption>2024年季度销售数据</caption>
  <thead>
    <tr>
      <th scope="col">季度</th>
      <th scope="col">销售额</th>
      <th scope="col">增长率</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">Q1</th>
      <td>100万</td>
      <td>10%</td>
    </tr>
  </tbody>
</table>
```

### 表格要求

| 元素 | 用途 |
|------|------|
| caption | 表格标题 |
| th | 表头单元格 |
| scope | 表头作用范围 |
| headers | 复杂表格关联 |

## 表单内容

### 标签

```html
<label for="email">邮箱地址</label>
<input type="email" id="email" name="email">
```

### 说明文本

```html
<label for="password">密码</label>
<input type="password" id="password" 
       aria-describedby="password-hint">
<p id="password-hint">密码需包含8位以上字符</p>
```

### 错误信息

```html
<label for="email">邮箱地址</label>
<input type="email" id="email" 
       aria-invalid="true"
       aria-describedby="email-error">
<p id="email-error" role="alert">
  请输入有效的邮箱地址
</p>
```

## 语义化标记

### 地标区域

| 元素 | 用途 |
|------|------|
| header | 页头 |
| nav | 导航 |
| main | 主内容 |
| aside | 侧边栏 |
| footer | 页脚 |

### 列表

```html
<!-- 无序列表 -->
<ul>
  <li>项目一</li>
  <li>项目二</li>
</ul>

<!-- 有序列表 -->
<ol>
  <li>步骤一</li>
  <li>步骤二</li>
</ol>

<!-- 定义列表 -->
<dl>
  <dt>术语</dt>
  <dd>定义</dd>
</dl>
```

## 动态内容

### 实时区域

```html
<!-- 重要通知 -->
<div role="alert">操作成功！</div>

<!-- 状态更新 -->
<div aria-live="polite">已加载 50 条记录</div>
```

### aria-live 值

| 值 | 用途 |
|-----|------|
| off | 不通知 |
| polite | 空闲时通知 |
| assertive | 立即通知 |

## 最佳实践

### ✅ 推荐
- 使用语义化 HTML
- 提供替代文本
- 结构化内容
- 测试屏幕阅读器

### ❌ 避免
- 无意义的链接文本
- 缺少图像替代文本
- 跳过标题层级
- 仅用视觉传达结构
