# 导航组件

## 概述

导航组件帮助用户在应用中定位和移动。良好的导航设计使用户能够理解当前位置、可去往的目的地，以及如何到达目标。

## 设计原则

- **清晰可见**：主要导航始终可见或易于访问
- **位置明确**：清晰指示当前位置
- **层级合理**：导航结构匹配信息架构
- **一致性**：导航模式在全应用保持一致

## 顶部导航栏（Header/Navbar）

### 尺寸规范

| 平台 | 高度 | 内边距 |
|------|------|--------|
| 网页端 | 64px | 24px |
| 移动端 | 56px | 16px |
| 桌面端 | 48-64px | 16px |

### 结构

```
┌────────────────────────────────────────────────────┐
│  Logo    导航项1  导航项2  导航项3     🔍  👤  ⚙️   │
└────────────────────────────────────────────────────┘
   左区域           中间区域              右区域
```

### 导航项状态

| 状态 | 样式 |
|------|------|
| Default | Gray-700，无下划线 |
| Hover | Gray-900，下划线 |
| Active | Primary-500，下划线或背景 |
| Disabled | Gray-400 |

|------|------|

| 移动端抽屉 | 280-320px |

### 导航项

| 元素 | 规范 |
|------|------|
| 高度 | 40-48px |

### 分组

- 使用分隔线或标题区分分组

- 支持折叠/展开子菜单

## 面包屑（Breadcrumb）

### 规范

| 元素 | 样式 |
|------|------|
| 分隔符 | `/` 或 `>` 或图标 |
| 间距 | 8-12px |
| 当前页 | 不可点击，深色 |
| 父级页 | 可点击，链接色 |

### 最佳实践

- 显示完整路径
- 过长时折叠中间层级
- 首项通常为"首页"
- 最后一项为当前页面

## 平台差异

### 网页端
- 顶部导航 + 侧边导航组合
- 支持悬停展开子菜单
- 面包屑显示完整路径
- 键盘可导航

### 移动端
- 底部导航为主
- 汉堡菜单展开侧边栏
- 标签页可滑动切换
- 面包屑可简化

### 桌面端
- 菜单栏设计
- 快捷键支持
- 右键上下文菜单
- 侧边导航可固定

## 无障碍考虑

- 使用语义化导航标签（`<nav>`）
- 当前页使用 `aria-current="page"`
- 可展开菜单使用 `aria-expanded`
- Tab 顺序符合视觉顺序
- 提供跳过导航链接

## 最佳实践

### ✅ 推荐
- 保持导航结构简洁
- 使用清晰的标签文案
- 明确指示当前位置
- 响应式适配不同屏幕

### ❌ 避免
- 导航层级过深（>3层）
- 标签文案过长
- 移动端使用悬停交互
- 隐藏重要导航入口
