# 设计原则

## 概述

设计原则是指导所有设计决策的核心理念。遵循这些原则，可确保产品在各平台上提供一致、优质的用户体验。本规范参考 Apple Human Interface Guidelines 的设计哲学，结合跨平台需求制定。

## 核心设计原则

### 1. 清晰（Clarity）

**让用户轻松理解界面内容和操作方式**

| 要素 | 实践指南 |
|------|----------|
| **文字** | 使用清晰易读的字体，保持足够的对比度 |
| **图标** | 图标含义明确，必要时配合文字说明 |
| **层次** | 通过视觉层次引导用户注意力 |
| **留白** | 合理使用留白，避免界面拥挤 |

#### 清晰原则的应用

```
✅ 推荐做法：
- 标题使用大字号和粗字重，与正文形成对比
- 主要操作按钮使用醒目颜色
- 分组信息使用分隔线或卡片区分

❌ 避免做法：
- 在小空间内堆砌过多信息
- 使用含义模糊的图标
- 低对比度的文字和背景
```

### 2. 一致性（Consistency）

**相同的元素在不同场景保持相同的外观和行为**

| 类型 | 描述 | 示例 |
|------|------|------|
| **视觉一致** | 相同元素外观统一 | 所有主按钮使用相同颜色 |
| **行为一致** | 相同操作产生相同结果 | 点击返回总是回到上一页 |
| **术语一致** | 相同概念使用相同词汇 | 统一使用"保存"而非混用"储存" |
| **平台一致** | 遵循平台设计惯例 | iOS 用手势返回，Android 用返回键 |

### 3. 高效（Efficiency）

**减少用户完成任务所需的步骤和时间**

#### 效率提升策略

| 策略 | 实现方式 |
|------|----------|
| **减少步骤** | 合并流程，移除不必要的确认 |
| **智能默认** | 预填合理的默认值 |
| **记住偏好** | 保存用户的选择和设置 |
| **快捷操作** | 提供键盘快捷键和手势 |
| **批量处理** | 支持多选和批量操作 |

### 4. 反馈（Feedback）

**每个操作都应有即时、清晰的反馈**

| 反馈类型 | 时机 | 示例 |
|----------|------|------|
| **即时反馈** | 操作触发时 | 按钮按下变色 |
| **过程反馈** | 操作进行中 | 加载进度条 |
| **结果反馈** | 操作完成后 | 成功/失败提示 |
| **状态反馈** | 状态变化时 | 网络断开提示 |

### 5. 容错（Forgiveness）

**允许用户犯错，并提供恢复方法**

```
容错设计要点：
├── 预防错误 .............. 禁用无效选项，验证输入格式
├── 确认危险操作 .......... 删除前确认，重要操作二次验证
├── 支持撤销 .............. 提供撤销/重做功能
├── 清晰的错误信息 ........ 说明错误原因和解决方法
└── 保存草稿 .............. 自动保存，防止数据丢失
```

### 6. 可访问（Accessible）

**确保所有用户都能使用产品**

| 考虑因素 | 设计要求 |
|----------|----------|
| **视觉** | 足够的对比度，支持缩放，不依赖颜色传递信息 |
| **听觉** | 为音频提供字幕，视觉替代方案 |
| **运动** | 支持键盘操作，足够大的点击区域 |
| **认知** | 简洁的语言，清晰的结构，一致的导航 |

## 设计决策框架

当面临设计决策时，按以下优先级考虑：

```
优先级从高到低：
1. 可用性 ................ 用户能否完成任务？
2. 可访问性 .............. 所有用户都能使用吗？
3. 清晰性 ................ 用户能理解吗？
4. 一致性 ................ 符合用户预期吗？
5. 效率 .................. 能否更快完成？
6. 美观性 ................ 视觉体验如何？
```

## 平台差异

### 网页端
- 考虑鼠标悬停状态（hover）
- 支持键盘快捷键
- 右键菜单上下文操作

### 移动端
- 优先考虑触控操作
- 适应不同屏幕尺寸
- 遵循 iOS/Android 平台规范

### 桌面端
- 充分利用大屏幕空间
- 菜单栏和工具栏设计
- 多窗口协作支持

## 无障碍考虑

- 所有交互元素可通过键盘访问
- 提供足够的颜色对比度（至少 4.5:1）
- 为非文本内容提供替代描述
- 支持屏幕阅读器

## 最佳实践

### ✅ 推荐

- 在开始设计前明确用户目标
- 保持界面简洁，突出核心功能
- 使用用户熟悉的模式和术语
- 提供清晰的视觉层次结构
- 测试真实用户的使用体验

### ❌ 避免

- 为了美观牺牲可用性
- 隐藏重要功能
- 使用不一致的设计模式
- 忽视边缘情况和错误处理
- 假设所有用户都是专家
