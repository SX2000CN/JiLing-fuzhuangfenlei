# 前端设计规范

> 参考 Apple Human Interface Guidelines 构建的跨平台前端设计规范体系

## 📖 规范概述

本设计规范旨在为网页端（Web）、移动端（Mobile）和桌面端（Desktop）提供一致、高质量的用户界面设计指导。通过遵循本规范，团队可以创建出清晰、直观、易用的数字产品。

## 🎯 设计愿景

**以人为本，追求卓越**

- **清晰（Clarity）**：文字清晰易读，图标精确明了，装饰恰到好处
- **尊重（Deference）**：界面服务于内容，流畅的交互帮助用户理解内容
- **深度（Depth）**：视觉层次和逼真的动效传达活力，便于理解和增添愉悦感

## 📁 文件命名规范

### 命名规则

所有规范文件采用统一的命名格式，通过文件名即可清晰了解文件的功能、模块归属和层级关系：

```
规范_{层级}_{模块}_{子模块}.md
```

- **层级**：两位数数字，表示文件在规范体系中的层级顺序
- **模块**：描述文件所属的主要模块
- **子模块**：描述文件的具体内容

### 命名示例

| 文件名 | 说明 |
|--------|------|
| `规范_00_概述_设计规范概述.md` | 第0层级，概述模块，设计规范概述内容 |
| `规范_01_设计基础_颜色规范.md` | 第1层级，设计基础模块，颜色规范内容 |
| `规范_02_组件规范_按钮.md` | 第2层级，组件规范模块，按钮组件内容 |

## 🔧 如何使用本规范

1. **通读设计原则**：首先理解核心设计理念
2. **查阅相关规范**：根据具体需求查阅对应规范文件
3. **遵循平台指南**：针对目标平台参考平台适配规范
4. **确保无障碍**：始终考虑无障碍访问需求
5. **使用资源文件**：根据需要使用提供的字体资源和示例代码
