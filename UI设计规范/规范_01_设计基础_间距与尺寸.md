# 间距与尺寸

## 概述

间距系统定义了界面元素之间的空间关系，合理的间距能够创造视觉节奏、区分内容层级、提升可读性。本规范基于 4px 基准建立完整的间距体系。

## 设计原则

- **一致性**：使用固定的间距值，避免随意数值
- **节奏感**：通过间距变化创造视觉韵律
- **亲密性**：相关元素间距更小，无关元素间距更大
- **呼吸感**：留出足够空白，避免拥挤

## 间距系统

### 基准单位

**基准值：4px**

所有间距值均为 4 的倍数，确保像素对齐和视觉一致。

### 间距比例

| Token | 值 | 像素 | 用途 |
|-------|-----|------|------|
| `space-0` | 0 | 0px | 无间距 |
| `space-0.5` | 0.125rem | 2px | 微小间距 |
| `space-1` | 0.25rem | 4px | 紧密元素 |
| `space-2` | 0.5rem | 8px | 相关元素内部 |
| `space-3` | 0.75rem | 12px | 元素内边距 |
| `space-4` | 1rem | 16px | **标准间距** |
| `space-5` | 1.25rem | 20px | 组件间距 |
| `space-6` | 1.5rem | 24px | 区块内边距 |
| `space-8` | 2rem | 32px | 区块间距 |
| `space-10` | 2.5rem | 40px | 大区块间距 |
| `space-12` | 3rem | 48px | 章节间距 |
| `space-16` | 4rem | 64px | 页面级间距 |
| `space-20` | 5rem | 80px | 大型留白 |
| `space-24` | 6rem | 96px | 页面级大留白 |

## 间距应用场景

### 内边距（Padding）

| 组件 | 内边距 |
|------|--------|
| 按钮（小） | 8px 12px |
| 按钮（中） | 10px 16px |
| 按钮（大） | 12px 24px |
| 输入框 | 12px 16px |
| 卡片 | 16px - 24px |
| 模态框 | 24px |
| 页面容器 | 24px - 32px |

### 外边距（Margin）

| 元素关系 | 间距 |
|----------|------|
| 相关元素（如标签与输入框） | 4px - 8px |
| 表单字段之间 | 16px - 24px |
| 段落之间 | 16px |
| 卡片之间 | 16px - 24px |
| 区块之间 | 32px - 48px |
| 页面章节之间 | 48px - 64px |

### 槽宽（Gap）

用于 Flex/Grid 布局：

| 场景 | 间距 |
|------|------|
| 图标与文字 | 4px - 8px |
| 按钮组 | 8px - 12px |
| 表单项 | 16px - 24px |
| 卡片网格 | 16px - 24px |
| 栅格槽宽 | 24px |

## 组件尺寸

### 高度规范

| 尺寸 | 高度 | 适用组件 |
|------|------|----------|
| **xs** | 24px | 小标签、紧凑按钮 |
| **sm** | 32px | 小按钮、表单元素紧凑模式 |
| **md** | 40px | **标准按钮、输入框** |
| **lg** | 48px | 大按钮、强调输入框 |
| **xl** | 56px | 超大按钮、移动端输入框 |

### 触控区域

| 平台 | 最小尺寸 | 推荐尺寸 |
|------|----------|----------|
| 移动端 | 44 × 44px | 48 × 48px |
| 网页端 | 32 × 32px | 40 × 40px |
| 桌面端 | 24 × 24px | 32 × 32px |

### 宽度规范

| 元素 | 宽度 |
|------|------|
| 小按钮 | 自适应，最小 64px |
| 标准按钮 | 自适应，最小 80px |
| 输入框（短） | 120px - 200px |
| 输入框（中） | 240px - 320px |
| 输入框（长） | 100%（最大 560px）|
| 模态框（小） | 400px |
| 模态框（中） | 560px |
| 模态框（大） | 800px |

## 圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `rounded-none` | 0px | 无圆角 |
| `rounded-sm` | 2px | 细微圆角 |
| `rounded` | 4px | **默认圆角** |
| `rounded-md` | 6px | 按钮、输入框 |
| `rounded-lg` | 8px | 卡片 |
| `rounded-xl` | 12px | 大卡片、模态框 |

| `rounded-3xl` | 24px | 大型容器 |

## 平台差异

### 网页端
- 使用 `rem` 单位支持整体缩放
- 响应式调整间距（移动端更紧凑）
- 最小宽度考虑 320px 视口

### 移动端
- 更大的触控区域（最小 44px）
- 相对紧凑的间距（节省屏幕空间）
- 考虑安全区域（Safe Area）

### 桌面端
- 可使用更宽松的间距
- 精确的鼠标交互允许更小目标
- 多窗口时考虑窗口最小尺寸

## 无障碍考虑

- 触控目标至少 44×44px（移动端）
- 可点击元素间距至少 8px
- 放大 200% 后布局不破坏
- 避免元素过于紧凑导致误触

## 最佳实践

### ✅ 推荐
- 严格使用间距 Token，避免魔法数字
- 相关元素使用较小间距
- 不同区块使用较大间距区分
- 保持垂直节奏一致

### ❌ 避免
- 随意使用间距值（如 13px、17px）
- 元素过于拥挤
- 不同场景间距不一致
- 忽视触控区域最小尺寸
