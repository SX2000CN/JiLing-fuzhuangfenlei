# 按钮

## 概述

按钮（Button）是最常用的交互组件，用于触发操作或提交表单。按钮应当清晰传达操作意图，并提供适当的视觉反馈。

## 设计原则

- **清晰明确**：按钮文案准确描述操作
- **主次分明**：不同重要程度使用不同样式
- **反馈及时**：点击后立即响应
- **位置一致**：相同操作的按钮位置保持一致

## 按钮类型

### 按重要程度分类

| 类型 | 用途 | 视觉特征 |
|------|------|----------|
| **主要按钮（Primary）** | 页面主操作，如提交、确认 | 实心填充，主色背景 |
| **次要按钮（Secondary）** | 次要操作，如取消、返回 | 边框样式，透明背景 |
| **文字按钮（Text）** | 低优先级操作，如查看更多 | 无边框，仅文字 |
| **危险按钮（Danger）** | 破坏性操作，如删除 | 红色填充或边框 |
| **幽灵按钮（Ghost）** | 轻量操作，工具栏 | 透明背景，悬停显示 |

### 按内容分类

| 类型 | 组成 | 用途 |
|------|------|------|
| 纯文字 | 文字 | 标准按钮 |
| 图标+文字 | 左图标+文字 | 增强语义 |
| 纯图标 | 图标 | 工具栏、紧凑空间 |
| 文字+图标 | 文字+右图标 | 下拉、链接跳转 |

## 状态定义

### 主要按钮状态

| 状态 | 背景色 | 文字色 | 边框 | 其他 |
|------|--------|--------|------|------|
| Default | Primary-500 | White | 无 | - |
| Hover | Primary-600 | White | 无 | cursor: pointer |
| Focus | Primary-500 | White | 无 | 焦点环 |
| Active | Primary-700 | White | 无 | - |
| Disabled | Gray-200 | Gray-400 | 无 | cursor: not-allowed |
| Loading | Primary-500 | White | 无 | 显示加载动画 |

### 次要按钮状态

| 状态 | 背景色 | 文字色 | 边框 |
|------|--------|--------|------|
| Default | Transparent | Gray-700 | Gray-300 |
| Hover | Gray-50 | Gray-900 | Gray-400 |
| Focus | Transparent | Gray-700 | Primary-500 |
| Active | Gray-100 | Gray-900 | Gray-400 |
| Disabled | Transparent | Gray-300 | Gray-200 |

## 焦点状态

所有按钮在获得键盘焦点时显示焦点环：

```css
/* 焦点环样式 */
outline: 2px solid Primary-500;
outline-offset: 2px;
```

| 尺寸 | 按钮大小 | 图标大小 |

### 使用指南

- 为图标按钮提供 `aria-label`
- 悬停时显示 Tooltip 说明
- 避免仅使用图标传达复杂操作

## 平台差异

### 网页端
- 支持 hover、focus 等丰富状态
- 使用 `<button>` 标签，支持键盘操作
- 可使用快捷键触发

### 移动端
- 无 hover 状态，使用 active 反馈
- 最小触控区域 44×44px
- 考虑使用全宽按钮
- iOS/Android 可使用系统触觉反馈

### 桌面端
- 参照网页端规范
- 支持键盘快捷键
- 工具栏按钮可更紧凑

## 无障碍考虑

- 使用语义化 `<button>` 标签
- 提供清晰的焦点指示
- 禁用状态使用 `disabled` 属性
- 图标按钮必须有 `aria-label`
- 加载状态使用 `aria-busy="true"`
- 颜色对比度 ≥ 4.5:1

## 最佳实践

### ✅ 推荐
- 按钮文案使用动词，如"提交"、"保存"
- 主要操作使用主要按钮，每个视图最多一个
- 破坏性操作使用危险按钮并二次确认
- 加载状态禁用按钮防止重复提交

### ❌ 避免
- 使用模糊的文案如"确定"、"是"
- 同一视图多个主要按钮
- 按钮过多导致选择困难
- 禁用状态不提供原因说明
