# 布局与网格系统

## 概述

布局系统定义了界面元素的组织方式，网格系统提供了一致的对齐和间距规则。良好的布局设计能帮助用户快速理解信息层次，高效完成任务。

## 设计原则

- **对齐**：元素沿网格线对齐，创造视觉秩序
- **一致**：相同类型的内容使用相同的布局模式
- **适应**：布局能适应不同屏幕尺寸
- **呼吸**：合理的留白让内容更易消化

## 栅格系统

### 基础栅格

采用 **12列栅格系统**，适应各种布局需求：

| 参数 | 值 | 说明 |
|------|-----|------|
| 列数 | 12 | 可被 2、3、4、6 整除 |
| 槽宽（Gutter） | 24px | 列之间的间距 |
| 边距（Margin） | 24px | 内容区与屏幕边缘距离 |

### 常用布局组合

| 布局 | 列分配 | 适用场景 |
|------|--------|----------|
| 单栏 | 12 | 文章详情、表单 |
| 双栏（1:1）| 6 + 6 | 对比展示 |
| 双栏（1:2）| 4 + 8 | 侧边栏 + 主内容 |
| 双栏（1:3）| 3 + 9 | 导航 + 主内容 |
| 三栏 | 4 + 4 + 4 | 卡片列表 |
| 四栏 | 3 + 3 + 3 + 3 | 特性展示 |

## 响应式断点

### 断点定义

| 断点名称 | 宽度范围 | 典型设备 | 栅格调整 |
|----------|----------|----------|----------|
| **xs** | < 576px | 小屏手机 | 4列，边距16px |
| **sm** | 576-767px | 大屏手机 | 4列，边距16px |
| **md** | 768-991px | 平板竖屏 | 8列，边距24px |
| **lg** | 992-1199px | 平板横屏/小桌面 | 12列，边距24px |
| **xl** | 1200-1399px | 桌面 | 12列，边距32px |
| **xxl** | ≥ 1400px | 大桌面 | 12列，边距32px |

### 内容最大宽度

| 场景 | 最大宽度 | 说明 |
|------|----------|------|
| 阅读内容 | 680px | 最佳阅读宽度 |
| 表单 | 560px | 单列表单 |
| 标准页面 | 1200px | 常规内容页 |
| 宽屏页面 | 1400px | 数据密集型 |
| 全宽 | 100% | 仪表板、管理后台 |

## 布局模式

### Flex 布局

适用于一维排列场景：

| 场景 | 方向 | 对齐方式 |
|------|------|----------|
| 顶部导航 | row | space-between |
| 表单字段 | column | stretch |
| 按钮组 | row | flex-start |
| 卡片列表 | row wrap | flex-start |

### Grid 布局

适用于二维复杂布局：

| 场景 | 列定义 | 间距 |
|------|--------|------|
| 卡片网格 | repeat(auto-fill, minmax(280px, 1fr)) | 24px |
| 仪表板 | 自定义 | 16px |
| 图片墙 | repeat(3, 1fr) | 8px |

## 平台差异

### 网页端
- 使用 CSS Grid 和 Flexbox
- 支持响应式断点全覆盖
- 考虑最小宽度 320px

### 移动端
- 主要使用 4 列栅格
- 边距固定 16px
- 内容优先全宽展示

### 桌面端
- 可使用更复杂的多栏布局
- 支持可调整的分栏宽度
- 考虑多显示器场景

## 无障碍考虑

- 内容顺序符合逻辑阅读顺序
- 页面缩放至 200% 仍可正常使用
- 避免横向滚动（移动端）
- 提供跳过导航链接

## 最佳实践

### ✅ 推荐

- 使用栅格系统保持对齐
- 为不同屏幕尺寸设计布局
- 内容区域设置最大宽度
- 保持一致的间距节奏

### ❌ 避免

- 过于复杂的嵌套布局
- 固定像素宽度（应使用相对单位）
- 忽略小屏幕适配
- 内容过于拥挤
