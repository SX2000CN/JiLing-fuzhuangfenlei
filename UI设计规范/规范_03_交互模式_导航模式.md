# 导航模式

## 概述

导航模式定义了用户在应用中移动和定位的方式。良好的导航设计帮助用户理解应用结构、找到所需内容、完成目标任务。

## 设计原则

- **可发现**：用户能找到导航入口
- **可理解**：导航结构清晰易懂
- **可预测**：导航行为符合预期
- **高效**：最少步骤到达目标

## 导航结构类型

### 层级导航（Hierarchical）

```
首页
├── 分类A
│   ├── 子分类A1
│   └── 子分类A2
└── 分类B
    └── 子分类B1
```

**适用场景**：内容层级分明的应用

**特点**：
- 逐级深入
- 支持返回上级
- 面包屑辅助定位

### 扁平导航（Flat）

```
┌────┬────┬────┬────┐
│ 首页│ 发现│ 消息│ 我的│
└────┴────┴────┴────┘
```

**适用场景**：功能并列的应用

**特点**：
- 顶级入口平等
- 快速切换
- 适合移动端底部导航

### 内容驱动导航

**适用场景**：内容消费类应用

**特点**：
- 内容即导航
- 推荐算法驱动
- 无限滚动

## 导航模式

### 标签页导航

| 平台 | 位置 | 数量 |
|------|------|------|
| 移动端 | 底部 | 3-5个 |
| 网页端 | 顶部 | 5-8个 |
| 桌面端 | 顶部/侧边 | 灵活 |

### 侧边栏导航

| 状态 | 宽度 | 适用 |
|------|------|------|
| 展开 | 240-280px | 桌面端 |
| 收起 | 64-72px | 桌面端 |
| 抽屉 | 280-320px | 移动端 |

### 汉堡菜单

**使用场景**：
- 移动端隐藏导航
- 次要导航入口
- 空间受限时

**注意**：
- 降低导航可发现性
- 增加操作步骤
- 仅在必要时使用

## 页面过渡

### 过渡类型

| 类型 | 动画 | 场景 |
|------|------|------|
| 前进 | 从右滑入 | 进入下级页面 |
| 后退 | 从左滑入 | 返回上级页面 |
| 替换 | 淡入淡出 | 同级切换 |
| 模态 | 从下滑入 | 临时任务 |

### 过渡时长

| 类型 | 时长 |
|------|------|
| 快速 | 150-200ms |
| 标准 | 250-300ms |
| 慢速 | 350-400ms |

## 返回导航

### 返回按钮

| 平台 | 位置 | 样式 |
|------|------|------|
| iOS | 左上角 | 箭头+文字 |
| Android | 左上角 | 箭头图标 |
| 网页端 | 左上角 | 箭头图标 |

### 返回行为

- 返回上一页面
- 保持页面状态
- 关闭模态/抽屉
- 退出编辑模式

## 深度链接

### 定义

直接链接到应用内特定页面。

### 实现要点

- URL 结构清晰
- 支持分享链接
- 处理无效链接
- 登录状态处理

## 平台差异

### 网页端
- URL 驱动导航
- 浏览器前进/后退
- 支持多标签页
- 书签功能

### 移动端
- 手势返回（iOS 边缘滑动）
- 系统返回键（Android）
- 底部标签栏
- 全屏沉浸式

### 桌面端
- 菜单栏导航
- 快捷键支持
- 多窗口管理
- 侧边栏固定

## 无障碍考虑

- 提供跳过导航链接
- 当前页面 `aria-current`
- 键盘可导航
- 焦点管理
- 屏幕阅读器支持

## 最佳实践

### ✅ 推荐
- 保持导航结构简洁（≤3层）
- 提供多种导航方式
- 明确指示当前位置
- 支持快速返回首页

### ❌ 避免
- 导航层级过深
- 隐藏重要导航
- 死胡同页面
- 不一致的导航模式
